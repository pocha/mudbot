# Mudbot

A self-hosted WhatsApp message automation platform. Connect your WhatsApp account via QR code, then schedule automated messages or send them on demand via a REST API.

All user data is stored encrypted in the repository — the only secret you manage is `SERVER_SECRET` in `.env`.

---

## How it works

- **Auth:** passwordless magic link sent to your email. Clicking the link gives you a session token.
- **Storage:** all user data lives under `users/` as AES-256 encrypted files. `tokens.json` is also encrypted. Only `.env` must be kept secret.
- **Scheduling:** cron jobs invoke `scripts/run-schedule.js` which decrypts and sends messages via [mudslide](https://github.com/robvanderleek/mudslide).

---

## Prerequisites

- Linux or macOS (Windows not supported)
- `curl` available on the system
- For development email: [MailDev](https://github.com/maildev/maildev) (`npm install -g maildev`)

---

## Installation

```bash
git clone <repo-url>
cd watobot
chmod +x install.sh
./install.sh
```

`install.sh` will:
1. Install Node.js + npm if not present
2. Install project dependencies (`npm install`)
3. Download and install the correct `mudslide` binary for your OS/arch to `/usr/local/bin/mudslide`
4. Create `.env` with a randomly generated `SERVER_SECRET` if one doesn't exist

---

## Configuration

Edit `.env` after installation:

```env
PORT=3000
SERVER_SECRET=<auto-generated — do not share>

# SMTP settings for magic link emails
SMTP_HOST=localhost
SMTP_PORT=1025
SMTP_SECURE=false
SMTP_USER=
SMTP_PASS=
EMAIL_FROM=noreply@mudbot.local
BASE_URL=http://localhost:3000
```

For **production**, point `SMTP_*` at a real mail provider (e.g. SendGrid, SES) and set `BASE_URL` to your public domain.

For **local development**, use MailDev as a local SMTP sink (see [Running tests](#running-tests)).

## Running the app

```bash
npm start
```

Server starts on `http://localhost:3000` (or `PORT` from `.env`).

Open `http://localhost:3000` in your browser to register and connect WhatsApp.

---

## Connecting WhatsApp

1. Register with your email — you'll receive a magic link
2. Click the link to open your dashboard
3. Click **Connect WhatsApp** to generate a QR code
4. Scan the QR code with WhatsApp on your phone (Linked Devices → Link a Device)
5. Once connected, you can create schedules or send messages via API

## API

All endpoints (except `/api/register` and `/api/verify/:token`) require authentication via:

```
Authorization: Bearer <token>
```
or
```
x-api-key: <api-key>
```

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/register` | Register with email, receive magic link |
| `GET` | `/api/verify/:token` | Verify token, get user info |
| `POST` | `/api/apikey/generate` | Generate a permanent API key |
| `GET` | `/api/whatsapp/status` | Check WhatsApp connection status |
| `GET` | `/api/whatsapp/qr` | Get QR code for WhatsApp login |
| `POST` | `/api/whatsapp/logout` | Disconnect WhatsApp |
| `POST` | `/api/message` | Send a message immediately |
| `GET` | `/api/schedules` | List all schedules |
| `POST` | `/api/schedules` | Create a schedule |
| `GET` | `/api/schedules/:id` | Get a schedule |
| `PUT` | `/api/schedules/:id` | Update a schedule |
| `DELETE` | `/api/schedules/:id` | Delete a schedule |
| `GET` | `/api/schedules/:id/logs` | Get execution logs for a schedule |

### Send a message

```bash
curl -X POST http://localhost:3000/api/message \
  -H "x-api-key: <your-api-key>" \
  -H "Content-Type: application/json" \
  -d '{"to": "919876543210@s.whatsapp.net", "message": "Hello!"}'
```

### Create a schedule

```bash
curl -X POST http://localhost:3000/api/schedules \
  -H "x-api-key: <your-api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Daily reminder",
    "recipients": ["919876543210@s.whatsapp.net"],
    "message": "Good morning!",
    "cronExpression": "0 8 * * *"
  }'
```

## Running tests

Tests require MailDev running locally to capture registration emails.

```bash
# Terminal 1 — start MailDev
npx maildev

# Terminal 2 — run tests
npm test
```

The test suite (`test/flow.test.js`) will:
- Start the server automatically if it isn't already running on port 3000
- Run through the full user flow: register → verify → API key → schedule CRUD
- Assert that files on disk are encrypted (not plaintext)
- Test re-registration preserves the user directory
- Clean up the test user after completion

MailDev's web UI is available at `http://localhost:1080` during the run.

## Security model

| What | How |
|------|-----|
| `tokens.json` | Encrypted as a blob with `SERVER_SECRET` |
| Email in tokens | AES-256 encrypted with `SERVER_SECRET` |
| User data files (`schedule.json`, `api_key`) | AES-256 encrypted with `hash(email + SERVER_SECRET)` |
| WhatsApp session (`.mudslide/`) | Plaintext, gitignored |
| Logs (`logs.txt`) | Plaintext, gitignored |

The entire repository (including `users/` and `tokens.json`) can be committed to a public repo safely. Only `.env` must be kept private.

> **Rotating `SERVER_SECRET`** will invalidate all existing tokens and render all user data unreadable. Back up `.env` and do not rotate without a migration plan.
